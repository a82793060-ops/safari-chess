# مواصفات المرحلة ٢ — المسار المُبوّب (بيدق)

وثيقة تصميم مرجعية. اقرأها مرّة لفهم الهيكل والمنطق كاملًا قبل كتابة محتوى نقاط التحقّق.
نصوص المهام لا تُكتب هنا — تُملأ داخل هذه البنية.

## ١) الغرض
تحويل لبنات التعلّم المتناثرة (المدرسة، الافتتاحيات، التكتيكات، النهايات، اللعب) إلى **مسار واحد
متدرّج** بمنطق **بوّابة لينة**: لا أقفال، التقدّم مسموح دائمًا، لكن المسار يوجّه بلطف إلى المحطة التالية.
اللبنات جاهزة (مستويات البوت v27). المطلوب: طبقة التنظيم فوقها (الترتيب، الحالات، نقاط التحقّق، التقدّم).

## ٢) محطات المسار (٦، بالترتيب)
| # | المعرّف | العنوان | اللبنة المصدر | المجموعة |
|---|---|---|---|---|
| ١ | school | المدرسة — حركة القطع | وحدة المدرسة | أساس |
| ٢ | rules | القواعد والمات | قواعد المدرسة | أساس |
| ٣ | openings | مبادئ الافتتاح | الافتتاحيات | مهارة |
| ٤ | tactics | التكتيكات الأساسية | التكتيكات | مهارة |
| ٥ | endgames | النهايات الأساسية | النهايات | مهارة |
| ٦ | play | اللعب ضدّ البوت المتدرّج | مستويات البوت (v27) | تطبيق |

```js
const TRACK_ORDER = ["school", "rules", "openings", "tactics", "endgames", "play"];
```

## ٣) حالات المحطة (بلا قفل)
`completed` (اجتاز ~٧٠٪+ → شارة «أتقنت» ✓) · `recommended` (التالية المنطقية → تمييز بصري) ·
`available` (متاحة عادية). لا حالة «مقفلة». recommended توجيه لا حاجز.

## ٤) منطق «الموصى بها»
```js
function getRecommended(progress) {
  return TRACK_ORDER.find(id => progress[id]?.status !== "completed") ?? null; // null = اكتمل المسار
}
```
كل ما عدا المكتملة والموصى بها = available. لا تبعيات معقّدة — الترتيب الخطّي يكفي.

## ٥) نموذج بيانات التقدّم (محلي)
```js
progress = {
  school:   { status: "completed",   bestScore: 90 },
  rules:    { status: "recommended", bestScore: 55 },
  openings: { status: "available",   bestScore: 0  },
  // ...
}
```
`status` تُشتقّ عند العرض (قاعدة ٤ + سجل الإتمام) — لا تُخزَّن recommended صراحةً.
`bestScore` (٠-١٠٠) أعلى نتيجة نقطة تحقّق.
```js
function onCheckpoint(progress, id, score) {
  const p = progress[id];
  p.bestScore = Math.max(p.bestScore, score);
  if (score >= PASS_THRESHOLD) p.status = "completed";
  return progress; // الرسوب/التخطّي لا يمنع التقدّم
}
const PASS_THRESHOLD = 70; // رحيمة، محاولات غير محدودة
```

## ٦) نقاط التحقّق (٣-٥ مهام/محطة، عتبة ٧٠٪، محاولات غير محدودة، تلميح بعد كل خطأ)
| المحطة | النوع | المهام | الاجتياز |
|---|---|---|---|
| school | أسئلة تعرّف (حركة القطع/الرقعة) | ٥ | ٤/٥ |
| rules | «جد المات في نقلة» + قواعد خاصة | ٤ | ٣/٤ |
| openings | مبادئ (سيطرة/تطوير/أمان الملك، لا حفظ خطوط) | ٤ | ٣/٤ |
| tactics | ألغاز (شوكة/تثبيت/سيخ) | ٥ | ٤/٥ |
| endgames | وضعيات نهاية (وزير/رخ، مربّع الملك) | ٤ | ٣/٤ |
| play | الفوز على مستوى بوت محدّد | مباراة | فوز على الهدف |

بنية عنصر المهمة:
```js
{ id, type: "puzzle"|"quiz"|"position"|"match", fen, prompt:{ar,en}, answer, hint:{ar,en} }
```

## ٧) الواجهة
مؤشّر تقدّم علوي «أكملت N من ٦» + شريط · بطاقة لكل محطة تعكس حالتها الثلاث · دخول أي
available/recommended مباشرةً · بعد نجاح نقطة تحقّق: تحديث المؤشّر + شارة «أتقنت» + انتقال «الموصى بها» تلقائيًّا.

## ٨) خارج النطاق
نظام نقاط/شارات موسّع · مزامنة أجهزة/حساب · عرض ELO (داخلي فقط).

## ٩) ملاحظات التنفيذ
المنطق كله من `TRACK_ORDER` (مصدر حقيقة واحد) · لا أقفال/تبعيات · تخزين محلي (هيّئ افتراضيًّا عند
الإقلاع) · ارفع النسخة وحدّث CLAUDE.md عند اكتمال المرحلة.

**المُخرَج التالي بعد اعتماد الوثيقة:** كتابة نصوص نقاط التحقّق (المهام/الوضعيات/الحلول، باللغتين).
